Arquitetura de Backend e Ecossistema Digital

🌐 Contexto do Projeto

O ecossistema digital da Cidade de Refúgio (ONG sem fins lucrativos) é composto por uma arquitetura híbrida projetada para escala e gestão eficiente:

🏠 Landing Page Pública

Portal de apresentação institucional da organização, focado em captação de recursos e visibilidade do projeto social.

🔐 Dashboard Administrativo (React)

Aplicação restrita para ADMs, responsável pela gestão de alunos, uploads de documentos e controle de pertences através de integração direta com este Backend.

🖼️ Diagrama de Arquitetura do Backend

🌟 1. Visão Geral do Backend

O backend atua como o motor central que serve tanto a Landing Page quanto o Dashboard React. Ele utiliza uma arquitetura Serverless para gerir o ciclo de vida dos dados dos alunos, desde o processamento de imagens com OCR até a geração de relatórios analíticos.

🧩 2. Componentes da Arquitetura

🚪 2.1 Interface (API)

  • API Gateway: Ponto de integração para o Dashboard React.

⚡ 2.2 Processamento

• lbd-salva-docs • lbd-cadastro-alunos • lbd-pertences • lbd-gerador-pdf • lbd-leitura-docs • lbd-get-alunos

🔄 3. Fluxos de Dados Principais

📥 Ingestão e OCR

Dashboard React → API Gateway → Lambda → S3 (Raw) → Textract → SOT.

📄 Gestão de Alunos

O Dashboard consome as rotas GET/POST da API para sincronizar dados em tempo real com o Data Lake.

📝 4. Observações de Integração

  • AUTH O login no Dashboard React deve validar credenciais antes de liberar chamadas para a cidade_refugio_api.
  • CORS Configurar o API Gateway para permitir requests vindas do domínio da Landing Page e do Dashboard.
